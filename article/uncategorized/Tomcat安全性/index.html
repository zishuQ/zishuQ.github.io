准备工作

  1. 下载tomcat二进制分发版apache-tomcat-9.0.87.tar.gz，9.0版本是tomcat的Stable版本，支持jdk1.8版本及以上运行

  2. 解压出来的文件和目录，里面的文件是对tomcat在构建、使用的一些说明。重点关注几个目录

    目录 作用
    bin 这个目录包含了Tomcat的可执行文件，如启动脚本、关闭脚本等。可以在这个目录中找到用于管理Tomcat服务器的命令行工具。
    conf 这个目录包含了Tomcat的配置文件，如服务器配置、日志配置、Web应用程序配置等。可以在这个目录中修改Tomcat服务器的各种配置选项。
    logs 这个目录通常包含Tomcat服务器生成的日志文件。可以在这个目录中查看Tomcat的运行日志、访问日志等
    webapps 这是一个Web应用程序的目录，通常用于存放部署到Tomcat服务器上的Web应用程序。将WAR文件放入这个目录时，Tomcat会自动将其部署并运行。
    lib 这个目录包含了Tomcat服务器运行时所需的Java类库文件。这些文件通常是Tomcat服务器的依赖库，用于支持Tomcat的运行和功能。
    temp 这是一个临时目录，Tomcat在运行时会使用它来存放临时文件和缓存数据。
    work 这是Tomcat服务器的工作目录，用于存放Tomcat运行时生成的临时文件和JSP编译后的Java类文件。

  3. 在bin目录下，有启动、关闭tomcat的脚本。启动可以使用startup.sh或者使用catalina.sh传入命令，startup.sh本质上也是使用catalina.sh启动，只是更简单一些，而catalina.sh支持更多启动项和参数，有更多命令，是一个更底层的脚本。关闭可以使用shutdown.sh或者catalina.sh传入命令。这里我直接使用startup.sh启动tomcat

安全性问题

1、Tomcat启动安全配置

  1. tomcat默认管理控制台和管理账号密码
    避免tomcat管理控制台对外开放，完全禁用对manager应用程序的访问，删除tomcat-users.xml禁止管理控制账号。如果有需要开启manager应用程序的访问，应该限制访问的IP地址为内部可信的地址，管理控制账号应按照用户角色配置，避免不同用户间共享账号。

    在conf/tomcat-users.xml中配置用户名和密码

  2. 删除tomcat不使用的样例文件和doc文件

    删除Tomcat运行不需要的doc文件和样例文件等文件，这些文件对tomcat运行没有作用，增加了安全风险，使用tomcat应该删除这些冗余文件。
    删除tomcat的冗余目录及文件，例如：

    目录 作用
    tomcat/webapp/doc/* 该目录下是tomcat操作介绍文档
    tomcat/webapp/examples/* 该目录下是tomcat的程序样例文档
    tomcat/webapp/ROOT/* ROOT是默认根目录，即访问 localhost:8080/，包含有关WEB的通用访问配置等

    也可以使用管理界面进行删除

  3. tomcat的shutdown端口和Connector端口安全配置
    Tomcat提供一个默认监听端口8005，通过本地TCP/IP连接到该端口，发送shutdown请求命令，从而终止tomcat进程。如果黑客知道这个端口开启，很容易被用来关闭服务。Connector默认配置端口8080是http请求监听端口，8443默认是https请求监听端口，使用这些默认端口，很容易成为黑客的攻击对象，增加被攻击的风险。

  4. 以安全管理模式运行Tomcat
    通过-security参数启动tomcat，则tomcat以安全管理模式运行，会自动加载tomcat/conf/Catalina.policy安全策略文件，服务可以根据需要对tomcat应用程序读取系统环境变量和执行敏感操作等进行授权。

2、日志与文件目录配置操作

  1. 确保tomcat日志配置正确
    Tomcat的AccessLogValve用于创建日志文件，格式与标准的web服务器日志文件相同，设备应配置日志功能，对用户登录进行记录，记录内容包括用户登录使用的账号，登录是否成功，登录时间，以及远程登录时，用户使用的IP地址。

    属性 作用
    className 必须被设置成org.apache.catalina.valves.AccessLogValve。
    directory 存放日志文件的目录，可以是相对路径或者绝对路径,在tomcat下面有个logs文件夹，那里面是专门放置日志文件的，也可以修改为其他路径。确保该目录及文件只能由tomcat用户读写和执行。
    prefix 日志文件前缀。
    suffix 日志文件后缀。
    pattern 日志输出模式，common方式时，将记录访问源IP、本地服务器IP、记录日志服务器IP、访问方式、发送字节数、本地接收端口、访问URL地址等相关信息在日志文件中。日志记录的模式为”%h %l %u %t “%r” %s %b”，这是记录远程主机、远程用户、用户代理、请求方法、请求成功以及字节发送量等信息。
    fileDateFormat 用来格式化日期，例如：”yyyy-MM-dd”。
    resolveHosts 是否将IP地址解析为主机名。

  2. tomcat日志配置文件处理器和日志级别
    Tomcat的conf目录下有一个logging.properties文件用于配置日志处理器（handler）、日志打印级别和设置控制台。conf/logging.properties配置属性如下：

    handlers：定义日志处理器。

    org.apache.catalina.core.ContainerBase.[Catalina].[localhost].level：设置日志级别。

    org.apache.catalina.core.ContainerBase.[Catalina].[localhost].handlers：指定日志处理器。

    java.util.logging.ConsoleHandler.level：设置控制台
